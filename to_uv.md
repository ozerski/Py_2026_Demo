### «Миграция на uv за 60 секунд» 🚀

Если у вас уже есть проект, созданный классическим способом (с `venv` и `requirements.txt`), вот как перевести его на сверхзвуковые скорости:

#### Шаг 1: Установка uv (5 секунд)

Если uv еще не установлен в системе:

* **Windows:** `powershell -c "irm https://astral.sh/uv/install.ps1 | iex"`
* **macOS / Linux:** `curl -LsSf https://astral.sh/uv/install.sh | sh`

#### Шаг 2: Инициализация проекта (10 секунд)

В корне вашей папки с проектом выполните:

```bash
uv init

```

Эта команда создаст файл `pyproject.toml` — современный стандарт описания проекта в Python, который заменит разрозненные конфиги.

#### Шаг 3: Импорт зависимостей (15 секунд)

Вместо того чтобы переписывать всё вручную, заставьте `uv` прочитать ваш старый список библиотек:

```bash
uv add -r requirements.txt

```

**Что произойдет:** `uv` мгновенно просканирует файл, скачает пакеты (если их нет в кэше) и создаст файл `uv.lock`, который намертво зафиксирует состояние вашей рабочей среды.

#### Шаг 4: Проверка (20 секунд)

Попробуйте запустить ваш скрипт через `uv`, не активируя никаких окружений вручную:

```bash
uv run main.py

```


### Почему это круто?

1. **Больше никакого `pip freeze`:** Ваши зависимости теперь управляются автоматически в красивом файле `pyproject.toml`.
2. **Забываем про `activate`:** `uv run` сам знает, какое окружение использовать.
3. **Общий кэш:** Если вы создадите второй проект с теми же библиотеками, `uv` не будет их качать. Он просто «подцепит» их из глобального кэша за 0.1 сек.
